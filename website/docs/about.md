## Why I Built SreeBase

Database systems often feel like magic. You write a SQL or NoSQL query, and gigabytes of data are sorted, filtered, and returned in milliseconds. 

I built SreeBase to **demystify that magic**. I wanted to understand:

1. **How do parsers work?** SreeBase has a custom, hand-written Lexer and Parser for its unique bracketless query language.
2. **How does disk I/O work?** SreeBase uses a Bitcask-style storage engine (similar to Riak) with an append-only log and background compaction.
3. **How do drivers connect?** SreeBase features a custom TCP protocol with a length-prefixed binary header, and a native Python SDK (`reddybase`).
